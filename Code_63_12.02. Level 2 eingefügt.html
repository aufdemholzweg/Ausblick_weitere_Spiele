Die Veröffentlichung meines Holztechnikspiels erfolgte in mehreren überschaubaren Schritten über die Plattform GitHub. Ziel war es, aus einer lokalen Datei auf meinem Computer eine frei zugängliche Internetseite zu machen, die Studierende ohne Installation nutzen können. Zunächst erstellte ich mir auf GitHub einen eigenen Account mit dem Namen „aufdemholzweg“. GitHub kann man sich vereinfacht als eine Art Online-Speicher für Programmdateien vorstellen. Innerhalb dieses Accounts legte ich einen neuen Projektordner (ein sogenanntes Repository) mit dem Namen „Holzen“ an. Dieser Ordner dient als zentraler Ablageort für alle Dateien des Spiels. Wichtig war dabei, das Projekt öffentlich sichtbar einzustellen, damit Studierende später ohne Passwort darauf zugreifen können. Im nächsten Schritt speicherte ich den von der KI erzeugten Code als Datei mit dem Namen „index.html“ auf meinem Computer. Diese Bezeichnung ist notwendig, damit der Server erkennt, dass es sich um die Startseite der Anwendung handelt. Anschließend lud ich diese Datei in mein Repository „Holzen“ hoch. Damit war das Spiel zwar online gespeichert, jedoch noch nicht als Webseite nutzbar. Abschließend aktivierte ich in den Einstellungen des Projekts die sogenannte „Pages“-Funktion. Dadurch weist man GitHub an, die hochgeladene Datei automatisch als Internetseite darzustellen. Nach kurzer Zeit erhielt ich eine feste Webadresse. Über diesen Link ist das Spiel seitdem direkt im Browser aufrufbar und kann sowohl im Seminar als auch außerhalb der Lehrveranstaltung genutzt werden, ohne dass zusätzliche Programme installiert werden müssen.
